Matthew Bohnsack's MS Thesis

Making Creative Commons More Useful with Usage Management

This thesis proposes and implements an architecture that can be used to make Creative Commons more useful, by applying usage management.

Authors

Matthew Bohnsack (bohnsack@gmail.com)

Related Projects

During discussions held in UNM ECE informatics group research meetings, we have come up with at least three follow-on project ideas related to this work that might be used to by undergraduate and graduate students as a basis for their own projects:

1. Website to view, add, and/or edit CC license information embedded with XMP in various media: images, etc.

This project could start at the undergraduate level and possibly move to masters research, with the optional inclusion of more sophisticated features such as the creation of and reasoning over mashup documents. At the undergraduate level, ideas for functionality, in rough implementation order might include:

  • A basic Ruby on Rails site
  • Users and authentication so different people can sign up for and use the service.
  • Image upload, with images associated with a particular user, so that a user can only see his images and can't see other peoples images.
  • View a user's images in a list/grid or other collection and individually, if you "drill in" to a particular image.
  • Implement the rest of the CRUD operations on a user's uploaded images: Create (covered above), Read (covered above), Update (e.g., starting with edit of basic meta data like name), and Delete.
  • It needs to be easy for a user to "re-download" images they've uploaded.
  • Add the ability to view CC license information embedded in XMP: CC license type, attribution name, attribution URL.
  • Add the ability to change CC license information embedded in XMP.
  • Note that for both view and change XMP data in images, I'm currently using the mini_exiftool gem (http://miniexiftool.rubyforge.org/) that depends on the Perl-based exiftool (http://www.sno.phy.queensu.ca/~phil/exiftool/)
  • Add support for different media types. E.g., video files, PDF, etc.

2. Implementation and further exploration of the "CC+ Broker Service" concept

In my thesis, I'm just going to mention the possibility of something like the CC+ Broker Service, but further work could examine things like:

  • A real implementation with all the details that go with this.
  • Is it a single service provided that everyone can use or is it a protocol/API specification that anyone can use to implement their own CC+ Broker Service?
  • What does the payment interface look like?

3. Mapping some of these ideas onto something to do with mobile devices, perhaps with Rhomobile or similar

This is an idea that isn't fleshed out to well at the moment, but one that holds some promise. One idea is for a mobile phone application that automatically embeds XMP CC license information into photos and video. This would probably go much deeper than what is possible with Rhomobile.
